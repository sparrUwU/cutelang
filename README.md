# 🎀 CuteLang:3

> Самый кавайный функциональный язык программирования в мире!  
> *«Чистое функциональное программирование с милым синтаксисом»* ✨

🎀 Три принципа синтаксиса CuteLang:3

1. Каждая конструкция — каомодзи
Никаких if, lambda, let, function. Только смайлики:

Обычный язык ||	CuteLang
function	   || ((F_F)>>
if	         || (?_?)
then         ||	(T_T)
else	       || (E_E)
let	         || (^_^)~
lazy         ||	(z_Z)
force	       || (O_O)!
print	       || (^o^)>
read	       || <(^o^)

2. Префиксная запись (как в Лямбда-исчислении)
Всё записывается перед аргументами, в стиле λ-исчисления:

Обычный синтаксис ||	CuteLang
fact(5)	          || __ fact 5
2 + 3	            || __ :+ 2 3
x == 1            || __ B= x 1
head(list)        ||	__ head list

Применение всегда через __ (два подчёркивания):
__ функция аргумент

3. Выражения, а не инструкции
Всё — выражение, у всего есть значение. Нет return, нет присваивания. Только вычисление:

text
(?_?) __ B= x 1                 | Всё это выражение возвращает значение
(T_T) 1                         | Возвращает 1
(E_E) __ :* x _ fact __ :- x 1  | Возвращает x * fact(x-1)

## 📋 О проекте

**CuteLang:3** — это эзотерический функциональный язык программирования, разработанный
в рамках курсового проекта по функциональному программированию. Язык основан на
лямбда-исчислении и реализован на **F#** с использованием платформы **.NET 8.0**.

### Концепция

Каждая синтаксическая конструкция языка — это **каомодзи** (японский текстовый смайлик).
Код на CuteLang выглядит как эмоциональная переписка, но под капотом — строгое
лямбда-исчисление с замыканиями, каррированием и ленивыми вычислениями.
